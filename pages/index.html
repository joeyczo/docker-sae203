Choix des services adaptés

Nous avons choisi d’utiliser le web, car pour ce que nous voulions faire, c’est-à-dire une plateforme de jeu en ligne, Il offrait le meilleur équilibre entre accessibilité et fonctionnalités. En utilisant le web, nous pouvons atteindre un large public.

  • NodeJs : Node js nous a permis de créer un serveur facilement, nous connaissions déjà tous Javascript ce qui nous a permis d’aller plus vite sur notre projet
  • Javascript : Notre projet est entièrement gérer en javascript, nous avons du apprendre comment fonctionner les socket en javascript grâce à la libraire socket.io qui nous a permis une grande flexibilité entre le serveur et les clients
  • Html & css : Le choix d’utiliser HTML et css dans notre projet web était un choix évident, afin que nous ayons totalement la main sur notre interface, et que nous puissions l’afficher sur mobile et PC sans problème.

Déploiement et de tests

Introduction

Notre projet, Plateaux est un jeu de plateforme en ligne constituer de plusieurs mini-jeux, tel que le Dos, Simon, Qst et le Memory.

Déploiement

Description du processus de déploiement :

Lors de ce projet nous avons utilisé :

  • Docker, qui nous a permis de stocké comme un serveur notre projet.
  • GitHub, qui nous a permis de pouvoir partager nos fichiers facilement et simplement en plus du fait que l’on avait une sauvegarde constante.
  • Figma, qui nous avons utilisé pour pouvoir créer un plan de design pour les pages HTML, ainsi que les différents éléments telle que les cartes, ... .

Etapes du projet :

Nous avons premièrement créé un GitHub ou nous nous sommes organisé pour pouvoir créer les différents jeux présent sur la plateforme, nous avons ensuite créé le docker, pour pouvoir héberger notre projet, puis nous nous sommes lancé dans le projet en programmant les différents jeux.

Problème rencontrer lors du déploiement

Lors du déploiement nous avons eu quelques conflits lors du partage de fichier sur GitHub que nous avons su aisément résoudre.

Tests

Description des différents types de tests effectués :

Pour effectuer les tests nous avons jouer aux jeux pour pouvoir trouver des erreurs et bugs possibles, aussi pour avoir un œils extérieur sur nos erreurs nous avons demandé de l’aide auprès de nos camarades pour tester les jeux également et trouver des problèmes que nous n’aurions sans doute pas vu.

Résultats des tests :

  • Nous avons plusieurs problèmes durant cette SAE notamment le coté serveur/client
  • La plateforme fonctionne.
  • Nous avons su résoudre les bugs identifié grâce à de multiples test qui nous on permis de cerné précisément les problèmes.

Validation

Tous les objectifs que nous nous étions fixé durant cette SAE n’ont pas été réalisé par manque de temps, cependant nous ne comptons pas abandonné le projet maintenant. Il nous manque le système de point, pour pouvoir faire un classement, mais aussi le passage de jeux automatique, pour passer d’un jeu à l’autre dès la fin de ce dernier.

Nous sommes satisfait de nous pour le travail fourni avec le temps qui nous été donné, Le travail est complet.

Définition des besoins

Nous avons créer un espace collaboratif où nous avons noter toutes nos premières idées, et poser le pour et le contre pour les différentes technologies que nous aurions pu utiliser. Nous avons ensuite créer des maquettes, et dessiner tous nos assets, comme les cartes, les logos, …
